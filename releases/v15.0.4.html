Precondiciones
  • Fix de REBU-4989 desplegado en producción (PR #2749, cherry-pick a UAT/prod del hotfix #2747).
  • Empresa "Test" (company 21) en producción con permisos de administrador.
Setup
  1. Crear por API una planilla de Servicios Profesionales y un cargo vacante (`QA-4989-*`).
  2. Crear un colaborador SP por el wizard (UI) y asignarle el cargo existente en el Paso 2.
  3. Verificar la precondición: el cargo quedó OCUPADO (`isVacant=false`) tras asignar el SP.
Pasos
  1. Desvincular al SP por API (`people.terminatePerson`) con fecha de hoy.
  2. Consultar el estado del cargo por API (`positions.getById`).
Resultado esperadoTras desvincular al SP, el cargo se libera automáticamente y queda "Vacante" (`isVacant=true`, `collaborator=null`), disponible para reasignación — igual que en el resto de tipos de contratación. Evidencia adicional del fix: el cargo puede eliminarse por API (`deletePosition` responde 200), donde antes del fix devolvía 409 por seguir ocupado.
